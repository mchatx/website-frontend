These MChad docs help you learn and use the application/web client as a Translator and as a Listener. Navigation help throughout the application is shown through visual image guides in their respective section.

Getting Started

The usage of Web Client is straightforward whereas the application requires you to download their packages on your desired choice of operating system with no additional dependencies and softwares.

How to Use the Web Client

New Room

Web Client gives you access to New Room .
Fill in the form to get your own transation Room. After Submitted the form you have to wait till your form gets reviewed and cleared by our developer MonMon . After reviewing, if you meet the criteria you will be granted a token and a translation room.
This can then be accessed by listeners for viewing your live translations.

Archive Client

Archive Client offers you the ability to view live translations of past streams done by translators in their room. All you have to do is enter into their respective rooms.

Installation

Install on Windows

You can download the latest version of MChad by going to our downloads tab. Choose from x32 or x64 based on your version of windows.

To find if your version of Windows is 32-bit or 64-bit. Read this article here.

When you've completed the download, extract the contents of the zipfile at your desired location. After extracting all you have to do is run the windows client with .exe file

Install on Android

You can download the latest version of MChad by going to our downloads tab. Download and install the .apk file.

Before Installation remember to enable "Install from Unknown Sources" in your android settings. You can read on how its done here.

Install on Linux

You can download the latest version of MChad by going to our downloads tab. Download the .zip file and extract it at your desired location. After extracting run the .x86_64 file.

Before Installation remember to set your file to be an executable. You can read more on how to do this here.

Install on MacOS

You can download the latest version of MChad by going to our downloads tab. Download the zipfile and extract it at your desired location. After extracting run the .app file.

macOS version of MCHAD is an experimental build.

translator panel

Translator Panel is located in the bottom part of the window in translator mode The main idea of the UI is to minimalize the need to use mouse so the translator's hands can stay on the keyboard.

First Row contains the text field where you will input text.

Second Row contains the following elements :

  • Prefix : Automatically add fixed prefix.
  • Suffix : Automatically add fixed suffix.
  • Colour Code : colour for the colour of the text.
  • Colour Toggle : Toggle to put in colour code.
  • Outline : Pick colour for the custom outline colour.
  • Outline Toggle : Toggle to put in custom outline colour.
  • Local Prefix : Add prefix to the live chat bouncing like [EN], [JP], [ID], etc... Without including it in the database.

Third row contains Profile and Preset that will be discussed in the next part

  • Step 1: Select Second row elements. Set their values and press enter.
  • Step 2: Once your message is sent, the message input will be emptied and you can immediately write your next message.
  • Step 3: You can click an entry to call extra panel to quickly delete or edit an entry.
  • Step 4: The edit panel will automatically load the setting of the entry, you can quickly change the text colour and outline from saved preset using the dropdown menu.

profile and preset

The profile and preset control is located in the third row of the translator panel.

A profile is basically a set of fixed prefix, suffix, and colour code.
You can see which profile you're in and the other loaded profile in the profile list panel.
  • Add Profile : Adds a new profile.
  • Shift up : Shift the selected profile up in the list.
  • Shift Down : Shift the selected profile down in the list.
  • Remove Profile : Delete the selected profile from the profile list.
  • Manage Preset : Open the preset panel.
  • You can switch between profile using tab key or up and down arrow keys.
  • You can also jump to default profile using shift + tab.

Manage Preset

You can open preset panel by clicking the "Manage Preset" button. Preset is basically profile list saved locally in your machine that you can load anytime. The save preset button will save the currently selected profile in the preset list. The Public button is to swap between locally saved preset and the public preset saved in the database.

To add/delete prefix, just pick the prefix then click "add to list" to add to the profile list or "delete" button to delete the selected prefix

Rest Api

Get Room List

HTTP Request GET to

http://[mimimimiooon]/Room/?room=[room]&link=[link]
  • [room] => to search particular room name
  • [link] => to search particular stream link [VideoID]
Just use YT[video ID] the link will be automatically parsed to "https://www.youtube.com/watch?v=[videoID]"

Test command :

curl -X GET http://[mimimimiooon]/Room/

Only returns the top 20 query result JSON with format as follows :

  • Nick : Name of the room (string)
  • EntryPass : If the room is password protected (bool)
  • Empty : If the room is empty or not (bool)
  • StreamLink : Link to the stream of the current active session (string)

Get a Script from the Archive

HTTP Request GET to

http://[mimimimiooon]/Archive/
  • content-type : application/JSON or text/plain
  • payload/body :
    • [link] => archive link, you can get it from the archive list.
    • [pass] => needed in case if the script is password protected, it is in SHA256 hash.

Test Command :

curl -X POST -H "Content-Type: application/json" --data   http://[mimimimiooon]/Archive/

Only returns the top 20 query result JSON with format as follows :

  • Stime : Timestamp in milisecond (number).
  • Stext : Main TL message (string)
  • CC : Character colour in hex "rrggbb" format (string), only missing # for parsing.
  • OC : Outline colour in hex "rrggbb" format (string), only missing # for parsing.

Stream Live Translation

HTTP Request GET to

http://[mimimimiooon]/Listener/?room=[room]&pass=[password]
  • [room] => name of the room can be obtained from the room list.
  • [pass] => needed in case if the room is password protected, it is in SHA256 hash.

Test Command :

curl  -X GET http://[mimimimiooon]/Listener/?room=Testing

Use event Source listener in JS :

Then es.addeventlistener to parse the incoming data.

Types of incoming data

  • -> {} empty json for ping.
  • -> flag = Connect, content just a welcome to server stuff to confirm the connection.
  • -> flag = Timeout, if no activitiy in the Mchad server for 30 minutes.
  • -> flag = insert, if there's a new entry.
      • list contains
    • _id: id of the entry.
    • Stime: unix epoch milisecond when the entry is uploaded to the server.
    • Stext: string text for the translation.
    • CC: Font colour, string in hex "rrggbb" format.
    • OC: Outline colour , string in hex "rrggbb" format.
  • -> flag = update, if there's a change on an entry.

content is the same as [insert], just use _id to find the locally saved entry and overwrite.

Get Schedule

HTTP Request GET to

http://[mimimimiooon]/Schedule/?room=[room]&link=[link]&tags=[tags]
  • [room] => to search particular room name
  • [link] => to search particular stream link
  • [tags] => to search particular tags

Test Command :

curl -X GET http://[mimimimiooon]/Schedule/

Only returns at as follows :

  • Room : Name of the Room (string).
  • Link : Link to stream (string)
  • Note : Note for the stream (string)
  • Time : Time in milisecond since UNIX epoch (number)
  • Tag : List of tags associated to the schedule (string)
